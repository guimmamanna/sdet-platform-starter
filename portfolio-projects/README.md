# Additional SDET Projects

This folder expands the repository beyond the core starter and shows how the portfolio can grow into multiple high-signal SDET projects.

These are intentionally scoped as serious project tracks, not loose idea lists. Each one is designed to look credible to hiring managers and easy to continue building.

| Project | Focus | Primary Stack | Why it strengthens the portfolio |
| --- | --- | --- | --- |
| `mobile-device-automation` | Native and hybrid mobile quality engineering | Appium 2, pytest, BrowserStack or AWS Device Farm | Shows mobile coverage, device strategy, and cloud execution depth |
| `accessibility-quality-gates` | Shift-left accessibility automation | Playwright, axe-core, Lighthouse CI | Demonstrates inclusive quality and front-end partnership maturity |
| `data-pipeline-quality` | Data and analytics test automation | pytest, SQL, pandas, Great Expectations or Soda | Shows breadth beyond UI/API into platform and analytics validation |
| `microservices-reliability` | Contract, resilience, and non-functional quality | Pact, k6, Testcontainers, service virtualization | Shows systems thinking around distributed services and release safety |

Recommended portfolio order:

1. Keep `sdet-platform-starter` as the main recruiter landing repo.
2. Build one of these projects into a second standalone repo.
3. Use the remaining ones as roadmap-backed expansion tracks or pinned follow-up projects.
